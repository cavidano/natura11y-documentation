## Primary Navigation Component

The primary navigation component is a complete header menu system. It's made up of three preassigned areas: the logo, global action buttons, and the main menu. The main menu container includes an optional search field.

There are two distinct placements available for the main menu. The first menu placement is below the logo, spanning the full width of the page:

</TextBlock>

<PrimaryNavigation />

<TextBlock>

The second menu placement is inline with the logo, creating a slimmer navigation:

</TextBlock>

<PrimaryNavigation navClass="primary-nav--inline--lg" searchWidth="200px" />

<TextBlock>

Both menus use the same markup but include a different *modifier* on their parent selector.

In particular, `.primary-nav--below--{x}` places the menu below the logo. In contrast, `.primary-nav--inline--{x}` places the menu inline with the logo. Here, ***x*** equals its breakpoint modifier (e.g., *lg* for large). At this point the component changes layout from mobile to desktop.

Figure 1 (below) provides the entire HTML for the primary navigation component.

</TextBlock>

<TableLineDescBlock targetFigure="1">

<TableLineDescRow lines="1">

The `.primary-nav--{x}--{y}` block-level selector is required.

It's completed with two modifiers: where ***x*** indicates the menu position (below or inline), while ***y*** indicates the component's responsive breakpoint.

</TableLineDescRow>

<TableLineDescRow lines="3-17">

The `.primary-nav__logo` selector is required.

It contains the website's logo and home link.

When using a single color `svg` for the logo, the `data-logo="brand"` attribute placed on the `<a>` tag sets the svg fill color to currentColor. This is useful when a theme color utility is present on a parent element.

</TableLineDescRow>

<TableLineDescRow lines="19">

The `.primary-nav__menu` selector is required.

The `id` attribute equals the value of `aria-controls` from the associated menu button (found within the `.primary-nav__buttons` selector).

It is recommended to include `aria-label="{y}"`, where y provides a descriptive name for assistive technology.

</TableLineDescRow>

<TableLineDescRow lines="47-54">

The `.form-entry` selector (search bar) is optional.

The `role="search"` attribute is present for assistive technology.

It is recommended to include `aria-label="Search"`, which provides a descriptive name for assistive technology.

Learn about <a href="/docs/forms/#formEntries">form entries</a>.

</TableLineDescRow>

<TableLineDescRow lines="58">

The `.primary-nav__actions` selector is required.

It contains the website's menu button (on mobile viewports).

</TableLineDescRow>

<TableLineDescRow lines="60-65">

The `.mobile-menu-toggle` button is required.

It toggles the main menu on mobile viewports. This button is a direct child of `.primary-nav__actions`.

The `aria-label="Toggle Menu"` attribute provides a descriptive name for assistive technology.

The `data-toggle="collapse"` attribute is required.

The `data-target-toggle` attribute equals the value of the main menu's `id` attribute.

</TableLineDescRow>

<TableLineDescRow lines="67-77">

The `.button-group` selector is optional.

It contains actions buttons that should be available on every page (e.g. *Sign In*, *Translate*).

The `data-target-toggle` attribute equals the value of the main menu's `id` attribute.

Learn about the <a href="/docs/buttons/#theButtonGroup">button group</a>.

</TableLineDescRow>

</TableLineDescBlock>
