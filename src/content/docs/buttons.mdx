## The Button Selector

Buttons within Natura11y require a `.button` utility class on an `<a>` or `<button>` tag, shown in Figure 1. By default, `.button` selectors inherit text color and font size from their parent.

The example below is a generic `.button` selector with no other styles yet applied.

</TextBlock>

<FigureBlock
    figureNumber="1"
    visualExample={(
        <div class="text-align-center">
            <a class="button" href="#1">
                Button
            </a>
        </div>
    )}
    codeExample={(`
        <a class="button" href="#1">
            Button
        </a> \n
        <button class="button">
            Button
        </button>
    `)}
/>

<TextBlock>

## Full-width Buttons

For buttons that span the width of their parent, add the .width-100 <a href="#1">size utility class</a> (Figure 3).

Do not use `.display-block` for full-width buttons. Buttons have a display property of flex (for vertical centering text with icons). Using use `.display-block` changes the buttons display property from `flex` to `block`. This breaks the vertical centering of the icon within the button (See <a href="#1">buttons with text and icon</a> below).

</TextBlock>

<FigureBlock
    figureNumber="3"
    visualExample={(
        <button class="button width-100 theme-primary">
            Button
        </button>
    )}
    codeExample={(`
        <a class="button width-100 theme-primary" href="#1">
            Button
        </a>
    `)}
/>

<TextBlock>

## Outlined Buttons

Create a button with an outline instead of a fill. Add the outline modifier class `.button--outline` to the `.button` selector. The outline width is a <strong>2x</strong> multiple of your theme's body border width.

The outline color inherits the button's text color. You can change the color using our <a href="#1">text color utility classes</a>.

</TextBlock>

<FigureBlock
    figureNumber="4"
    visualExample={(
        <div class="grid grid--column-2--md gap-3">

                Button
            </a>
            <a class="button button--outline display-block text-color-link" href="#1">
                Button
            </a>
        </div>
    )}
    codeExample={(`
        <a class="button button--outline" href="#1">
            Button
        </a> \n
        <a class="button button--outline text-color-link" href="#1">

        </a>
    `)}

## Button Border Radius

Globally, all <code>.button</code> selectors have a border radius. This value is set with the <code>:root</code> scoped CSS Variable <code>--button-border-radius</code> (see Figure 5). Learn how to <a href="{{ site.url }}/docs/customization/#referenceGuideCSSVariables">customize CSS Variables</a>. 

</TextBlock>

<FigureBlock
    figureNumber="5"
    codeLanguage="css"
    visualExample={(
        <a class="button width-100 theme-primary" style={{'--button-border-radius': '0'}} href="#1">
            Button
        </a>
    )}
    codeExample={(`
        /* Globally remove border radius in CSS */ \n
        :root {
            --button-border-radius: 0;
        }
    `)}
/>

<Divider id="textAndIconButtons" />

<TextBlock>

## Text and Icon Buttons

Add an icon to a `.button` selector with the `.button--has-icon` modifier class (Figure 7). Buttons with icons require two child selectors.

The first child is is a `<span>` with its combined <a href="{{ site.url }}/icons">icon</a> classes. The second child is a `<span>` with a `.button__text` class. This holds the button's text. Further, the `.button__text` selector is our target for underlining the text on `:hover` and `:focus` states. Further still, it provides a small margin to separate the text from the icon.

</TextBlock>

<FigureBlock
    figureNumber="8"
    visualExample={(
        <div class="grid grid--column-2--md gap-3">
            <a class="button button--has-icon width-100 theme-primary" href="#1">
                <span class="icon icon-edit"></span>
                <span class="button__text">Edit</span>
            </a>  
            <a class="button button--has-icon width-100 button--outline" href="#1">
                <span class="icon icon-delete"></span>
                <span class="button__text">Delete</span>
            </a>
        </div>
    )}
    codeExample={(`
        <a class="button button--has-icon theme-primary" href="#1">
            <span class="icon icon-edit"></span>
            <span class="button__text">Button</span>
        </a> \n
        <a class="button button--has-icon button--outline" href="#1">
            <span class="icon icon-delete"></span>
            <span class="button__text">Delete</span>
        </a>
    `)}
/>

<TextBlock>

### Dispersing the text and icon

Create a button where the text and icon sit on opposing sides. Add the chain modifier class `--disperse` to `.button--has-icon` class. Chained together, the class is `.button--has-icon--disperse`. By default, this class makes the button full-width of its parent.

</TextBlock>

<FigureBlock
    figureNumber="9"
    visualExample={(
        <a class="button button--has-icon--disperse theme-dark" href="#1">
            <span class="button__text">Button</span>
            <span class="icon icon-arrow-right"></span>
        </a>
    )}
    codeExample={(`
        <a class="button button--has-icon--disperse theme-dark" href="#1">
            <span class="button__text">Button</span>
            <span class="icon icon-arrow-right"></span>
        </a>
    `)}

## Icon-only Buttons

For icon-only buttons, apply the `.button--icon--only` modifier class to the `.button` selector.

Icon-only buttons require a single child selector. A `<span>` with its combined icon classes (just like the text and icon buttons described above).

</TextBlock>

Do not use the <code>title</code> attribute to describe the button(on its hover state). Instead apply the <code>aria-label</code> attribute (see Figure 10). Icon-only buttons with the aria-label attribute offer a custom visible tooltip. The core
javascript file (<strong>natura11y.js</strong>) adds these tooltips to the DOM when the document loads.

</TextBlock>

<TextBlock>

## The Button Group

Need a horizontal list of buttons? You can place mixed button types (text and icon) within the `.nav` <a href="/docs/navigation">component</a>. You would need to include the `.nav--horizonal` modifier class. We designed the `.nav` component mostly for response text navigation. For legibility and cognitive accessibility, a healthy space is present between each item within the `.nav` component.

This is not always your best option for horizontal buttons. Particularly for grouping icon-only buttons (or mixed icon and text buttons) along the horizontal axis. 

The `.button-group` selector aligns buttons horizontally while giving them the same height. The value of `--button-icon-size` determines the height of `.button-group`. All buttons within `.button-group` stretch to fit this height.

</TextBlock>
