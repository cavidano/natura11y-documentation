## Layout Options

The article component offers layout options with two or three columns. With the example below, `.article--column-2` creates a two-column layout. Can you guess the class name needed for a three column layout?

The article component is made up of three preassigned areas: header, sidebar, and body. 

Instead of providing <a href="#1">breakpoint modifiers</a>, the layout changes from mobile to desktop view at the *large* viewport, and again at the *extra-large* viewport. The latter is true only for three column layouts. 

</TextBlock>

<TableLineDescBlock targetFigure="1">

<TableLineDescRow lines="1">

The parent `.article--column-{x}` selector is required.

Here ***x*** equals the amount of columns (either 2 or 3).

</TableLineDescRow>

<TableLineDescRow lines="3-5">

The `.article__header` selector is optional. 

It holds the article's `header` tag.

</TableLineDescRow>

<TableLineDescRow lines="7-9">

The `.article__sidebar` selector is required.

It holds the article's sidebar or navigation content.

</TableLineDescRow>

<TableLineDescRow lines="11-13">

The `.article__body` selector is required. 

It holds the article's main content.

</TableLineDescRow>

</TableLineDescBlock>
