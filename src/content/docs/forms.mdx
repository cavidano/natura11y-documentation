<TextBlock>

Forms are notoriously challenging to style and make accessible. Natura11y provides basic style and behavior for building user friendly, accessible forms. 

</TextBlock>

<AccessibilitySpotlight>

Form elements include a consistent visible focus for keyboard users.

Include accessible field labels <a href="https://www.w3.org/WAI/tutorials/forms/labels/#associating-labels-implicitly" target="_blank">implicitly</a> or <a href="https://www.w3.org/WAI/tutorials/forms/labels/#associating-labels-explicitly" target="_blank">explicitly</a>â€”it's up to you.

Intuitivly mark *required* fields. When doing so, Natura11y adds *required* and *data-required* attributes automatically.

Checkbox and radio groups use semantic *fieldset* and *legend* tags. 

</AccessibilitySpotlight>

## Form Entries

The form entry parent selector (`.form-entry`) wraps each entry within a form. Child elements within form entries vary depending on their input types. Such types include *text inputs*, *textareas*, *selects*, *checkboxes*, and *radios*.

In Figure 1, the `.form-entry` selector contains an `<input>` field with the `type="text"` attribute. 

</TextBlock>

<FigureBlock
    figureNumber="1"
    visualExample={(
        <FormEntry />
    )}
    highlightedLines="1,3-17,18-21"
    codeExample={(`
        <div class="form-entry"> \n
            <label class="form-entry__field"> \n
                <span class="form-entry__field__label">
                    Form Entry Label
                </span> \n
                <span class="form-entry__field__input">
                    <input
                        type="text"
                        name="textInputExample"
                        id="text-input-example"
                        aria-describedby="help-text-input-example">
                </span> \n
            </label> \n
            <small class="form-entry__help" id="help-text-input-example">
                Form entry help text
            </small> \n
        </div>
    `)}
/>

<TableLineDescBlock targetFigure="1">

<TableLineDescRow lines="1">

The `.form-entry` parent selector is required.

It contains each entries elements, including help text and feedback messages.

When more than one `.form-entry` selector is present, spacing between them occurs automatically.

</TableLineDescRow>

<TableLineDescRow lines="3-17">

The `.form-entry__field` child selector is required.

It holds the field's label and input elements.

The `.form-entry__field__label` grandchild selector is required.

It holds the actual label text, and required indicator (when present).

The `.form-entry__field__input` grandchild selector is required.

It holds the field's actual input elements.

When using help text, `aria-describedby` is present on the `input` tag for assistive technology. It equals the help text's `id` attribute (below).

</TableLineDescRow>

<TableLineDescRow lines="18-21">

The `.form-entry__help` child selector is not required.

Its id equals its associated `aria-describedby` attribute.

</TableLineDescRow>

</TableLineDescBlock>

## Select Element

With the example in Figure 2, the same `.form-entry` container holds a `<select>` element.

</TextBlock>

<FigureBlock
    figureNumber="2"
    visualExample={(
        <FormEntrySelect />
    )}
    highlightedLines="5-7,9-21"
    codeExample={(`
        <div class="form-entry"> \n
            <label class="form-entry__field"> \n
                <span class="form-entry__field__label">
                    Form Entry Label
                </span> \n
                <span class="form-entry__field__select">
                    <select
                        id="select-example"
                        name="selectExample"
                        aria-describedby="help-select-example">
                            <option value="">Select</option>
                            <option value="Option One">Option One</option>
                            <option value="Option Two">Option Two</option>
                            <option value="Option Three">Option Three</option>
                            <option value="Option Four">Option Four</option>
                            <option value="Option Five">Option Five</option>
                    </select>
                </span> \n
            </label> \n
            <small class="form-entry__help" id="help-select-example">
                Form entry help text
            </small> \n
        </div>
    `)}
/>

<TableLineDescBlock targetFigure="2">

<TableLineDescRow lines="5-7">

The `.form-entry__field__label` grandchild selector is required.

The `for` attribute equals its associated `<select>` tags `id`.

</TableLineDescRow>

<TableLineDescRow lines="9-21">

The `.form-entry__field__select` grandchild selector is required.

It holds the actual `<select>` tag.

The `<select>` tag's `id` equals its associated `for` attribute.

</TableLineDescRow>

</TableLineDescBlock>

## File Upload

Natura11y offers a simple file upload component, as shown in Figure 4 below.

</TextBlock>

<FigureBlock
    figureNumber="4"
    visualExample={(
        <FormEntryFileUpload />
    )}
    highlightedLines="11,13-15,17-22,24-27"
    codeExample={(`
        <div class="form-entry"> \n
            <div class="form-entry__field"> \n
                <span class="form-entry__field__label">
                    Upload a Single File
                </span> \n
                <div class="form-entry__field__input"> \n
                    <label class="file-upload"> \n
                        <span class="file-upload__drop">
                            <span class="file-upload__drop__text">Drag and Drop</span>
                        </span> \n
                        <input
                            class="file-upload__input"
                            type="file"
                            name="fileUploadExample"
                            id="file-upload-example"
                            accept="image/*" /> \n
                        <span class="button button--outline button--has-icon file-upload__button">
                            <span class="icon icon-upload"></span>
                            <span class="button__text">Browse for a File</span>
                        </span> \n
                    </label> \n
                </div> \n
            </div> \n
            <small class="form-entry__help" id="help-file-upload-example">
                Form entry help text
            </small> \n
        </div>
    `)}
/>

<TableLineDescBlock targetFigure="4">

<TableLineDescRow lines="11">

The `.file-upload` selector is required.

It holds the drop area and actual `<input>` tag.

</TableLineDescRow>

<TableLineDescRow lines="13-15">

The `.file-upload__drop` child selector is required.

It holds the `.file-upload__drop__text` selector.

</TableLineDescRow>

<TableLineDescRow lines="17-22">

The `.file-upload__input` child selector is required.

It includes the `type="file"` attribute.

</TableLineDescRow>

<TableLineDescRow lines="24-27">

The `.button` selector is required.

It can be styled with any of the <a href="/docs/buttons">button</a> utilities.

</TableLineDescRow>

</TableLineDescBlock>
